## AI量化编程该怎么学 ？

### 一、主题概述

今天我们聊一下，在 AI 这么发达的时代，量化交易当中的编程应该怎么选。

这里主要介绍的是量化策略和量化研究当中的编程技能，主要是 Python 相关技能。

在过去，如果没有进行 Python 量化交易编程，我们是需要自己写代码的。

而现在已经转变为：

1. 我们将自己的需求和思路告诉 AI
2. 让 AI 去写代码
3. 我们自己在 AI 生成代码的基础上进行修改

过去我们需要记住很多 Python 里的语法知识。

而现在，我们可能更加要注意：

1. 策略逻辑如何设计
2. 数据怎么样处理
3. 如何更好地利用 AI 帮助我们实现需求

下面将用一款经典的即时战略游戏《红色警戒》来打比方，分别介绍在初阶、中阶、高阶以及最终实盘阶段，每个阶段需要掌握哪些 Python 相关技能。

### 二、初级阶段：Python 基建工作

#### 1. 搭建 Python 环境

在初级阶段，我们首先需要做的是 Python 相关的基建工作。

这就像《红色警戒》里面，我们首先得把基地展开，把矿场和电厂搭起来。

如果没有电，没有矿场这些经济来源，后面的重工、高科技全部起不来。

所以这些都是基础中的基础。

对于 Python 来说，这一步包括：

1. 配置 Python 环境
2. 配置 IDE
3. 安装 AI 助手

现在比较经典的 IDE 有：

1. VS Code
2. PyCharm

比较常见的 AI 助手有：

1. Copilot
2. Claude Code
3. Codex

当你把 Python 环境、IDE 以及 AI 助手装好之后，就可以敲一行 Hello World。

如果 Hello World 运行成功，就说明你的基地已经展开，电厂和矿场都已经造好了。

#### 2. 学习 Python 基础语法

在 Python 的 AI 环境搭建好之后，第二步就是掌握一些 Python 的基础语法。

放在《红色警戒》里面，就像造一个兵营，出一些动员兵，出一些狗去探路，以及出一些工程师去占几个油田等等。

对应到 Python 里面，需要学习的基础语法包括：

1. 变量有哪些
2. 循环怎么写
3. 常用判断怎么写
4. 函数如何定义
5. 类的相关知识

#### 3. 掌握数据处理三件套

在掌握完 Python 的基础语法之后，接下来是重中之重。

我们需要掌握 Python 数据处理以及量化编程里的三件套：

1. pandas
2. NumPy
3. Matplotlib

这就像《红色警戒》里面造雷达和重工。

其中，数据处理里的 NumPy 和 pandas，就相当于重工，负责主力战斗。

而 Matplotlib 以及 Plotly 这两个库，就相当于雷达。

这两个库主要用于画图，就像雷达可以扫描地图一样，画图库可以绘制我们的净值曲线、回撤走势图等等。

### 三、中级阶段：写策略与做回测

#### 1. 写简单策略

当你把数据处理三件套掌握好之后，就需要用 pandas、NumPy 以及 Matplotlib 去写一些简单策略，并绘制策略的收益走势图。

这就像从重工里面出一些坦克，去打一场小的战斗。

对应到量化编程里面，就是使用上面的这些技能，包括数据处理包和基础语法，去写一个简单策略。

哪怕是最简单的双均线买卖策略，也已经能够上战场了。

#### 2. 评估策略表现

当你的坦克进行完一场战斗之后，就像你的策略已经跑完之后，你需要看看策略表现怎么样，也就是看看战绩怎么样。

在 Python 量化编程里面，这对应的就是回测和绩效评估。

我们需要知道：

1. 夏普比怎么算
2. 最大回撤怎么算
3. 净值怎么算
4. 收益率怎么算
5. 其他风险指标怎么算

这些风险指标常用的经典包包括：

1. empyrical
2. quantstats

所以在中级阶段，我们需要完成两件事：

1. 写一个简单策略
2. 利用相关工具验证策略收益表现如何

当你会写简单策略，并且会做一些简单回测之后，就可以进入高阶技能阶段。

### 四、高级阶段：根据方向学习专项技能

在高级阶段，根据方向不同，我们可能会有针对性地学习不同的 Python 编程技能。

#### 1. 股票多因子或期货多因子方向

如果你的方向是股票多因子，或者期货多因子相关，那么可能重点需要掌握 Alphalens 这个 Python 包。

所谓多因子，并不仅仅是利用单一指标，而是可以进行多个指标的相互组合。

这就像《红色警戒》里面：

1. 防守可以造碉堡、光棱塔以及大炮
2. 陆地部队可以造光棱塔和幻影坦克
3. 空军可以造黑鹰战机、基洛夫或者飞行兵等等

这是一种立体化的作战方式。

而多因子也是一样，它不仅仅只利用动量或者均线这样的技术指标因子，还可以利用：

1. 估值因子
2. 波动率因子
3. 宏观因子
4. 另类因子

通过综合因子，构建一个更加立体的策略。

在因子的清洗、构造、检验以及组合过程中，一个不可或缺的经典库就是 Alphalens。

所以多因子相关方向，重点需要掌握 Alphalens 这个包。

#### 2. 期货 CTA 策略方向

如果你做的是期货 CTA 策略，那么可能需要掌握 CTA 以及回测相关的一些库。

比如国内经典的 vn.py，以及回测框架的经典库：

1. Backtrader
2. vectorbt

CTA 策略基本上主要是趋势跟踪加止损纪律。

这就像《红色警戒》里面，当你发现敌人防线有缺口、有漏洞的时候，坦克带狗一波冲上去，掐住对方的头左右摇摆，就可以获得比较大的优势。

除了掌握经典的 vn.py、Backtrader 以及 vectorbt 这几个包之外，在编程逻辑相关方面，更加需要掌握：

1. 信号如何写
2. 信号如何触发
3. 怎么下单比较合理
4. 如何避免未来函数
5. 如何减少滑点
6. 如何写风控
7. 如何写仓位管理
8. 如何进行止损
9. 如何进行参数设置

这些都是业务上的逻辑。

#### 3. 固定收益或资产配置方向

如果你做的是固定收益或资产配置相关内容，那么个人认为可能用得比较多的是优化以及定价相关内容。

优化常用的包是 SciPy。

比如，当我们选择了一揽子标的之后，需要将资金在股票、商品、债券、现金以及其他资产之间，选择一个最优权重。

而这个最优权重怎么来，就是优化包需要做的事情。

如果把资产配置放到《红色警戒》里面，它就不仅仅是选择某一个单一阵营或者国家去战斗。

它更像邪恶联盟，既有苏军的科技，也有盟军的科技。

你需要根据战略目标，在不同技能树上选择合适的加点和匹配。

#### 4. 机器学习、深度学习与 AI 方向

机器学习、深度学习以及 AI 相关知识，在最近这些年用得越来越多，甚至已经成为一种常规操作库。

机器学习里面用得最多的是 scikit-learn。

scikit-learn 里面有很多经典的机器学习模型，但实际上没有必要全部掌握。

用得比较多的模型包括：

1. 线性回归
2. 聚类
3. 逻辑回归
4. 随机森林

在 scikit-learn 之外，用得比较多的是树模型，比如：

1. XGBoost
2. LightGBM
3. CatBoost

这几个树模型库用得也比较多。

此外还有一些深度学习模型。

如果要用深度学习，一般 PyTorch 用得比较多，TensorFlow 用得也比较多。

这些机器学习、深度学习以及 AI 相关应用，就相当于《红色警戒》里面的超级武器。

比如：

1. 铁幕
2. 超时空传送
3. 核弹
4. 闪电风暴

虽然这些叫超级武器，但实际上近些年来，这些技能越来越多地变成了常规应用手段。

### 五、最终阶段：对接实盘

当你把高级阶段中的某一个方向吃透之后，比如多因子方向或 CTA 方向，下面就可以对接实盘。

对接实盘就相当于我们打《红色警戒》时，可以上兰博玩或者战网，去和别人对战一样。

实盘也需要一个途径去对接真实交易。

这个途径就是对接券商柜台，或者期货公司柜台。

#### 1. 股票与 ETF 实盘对接

如果你是做股票，一般会用 QMT 以及 Ptrader 去对接券商柜台。

#### 2. 期货实盘对接

如果你是做期货，一般会用 CTA 接口去对接交易。

QMT 或 CTA 接口的一些语法知识，是需要掌握的内容。

比如：

1. 如何获取行情
2. 如何下单
3. 如何接收成交回报
4. 如何接收委托回报
5. 如何进行断线重连

在掌握行情以及交易 API 之后，更需要了解在编程里面如何做一个交易系统，以及实盘里的风控怎么写。

这些是实盘阶段需要掌握的一些 Python 量化编程技能。

4. 重点掌握 Python 数据处理三件套

Python 数据处理三件套包括：

1. pandas
2. NumPy

#### 2. 中级阶段

中级阶段需要做的事情包括：

1. 自己动手写一些简单策略

高级阶段需要在多因子、CTA、资产配置以及其他几个大方向中，重点先攻克一个方向。

## 量化策略英雄池

> 来源：用户上传文本 :contentReference[oaicite:0]{index=0}

### 一、分类维度

量化策略可以按照两个维度进行划分：

1. 波动大小
2. 择时或非择时

按照这两个维度，可以大致分为四个象限：

1. 波动大且择时的策略
2. 波动小且择时的策略
3. 波动小且非择时的策略
4. 波动大且非择时的策略

### 二、第一象限：波动大且择时的策略

第一象限是波动大且择时的策略。

这类策略的特点是特别看切入时机，也就是择时。

它就像 DOTA 里面的流浪剑客：

如果切入时间好，就可以把伤害拉满，也就是收益会非常大。

但如果切入时间不太行，就很容易团战打崩，也就是会面临比较大的投资亏损。

所以，这类策略虽然可能带来比较大的潜在收益，但对择时的要求比较高。

常见的择时且波动大的策略有：

1. CTA 策略
2. 宏观择时策略
3. 事件驱动策略

### 三、CTA 策略

CTA 策略如果用一个词语来形容，就是追涨杀跌。

它的本质是做趋势跟踪。

它认为，如果市场价格一旦形成趋势，无论这个趋势是上涨还是下跌，在短期或中期内，它都会继续延续这个方向。

所以，该策略的算法会通过识别趋势起点，在趋势启动的时候进行跟进，在趋势即将反转的时候进行离场。

#### 1. CTA 策略的特点

##### 可以做多，也可以做空

一般 CTA 策略是在期货市场，而期货市场可以双向交易。

所以，无论市场上涨还是下跌，只要判断对了方向，都能够盈利赚钱。

##### 通常会做多品种分散

对于市场行情来说，大部分时间都是震荡的。

只有极少数时间，市场会形成一波比较强的趋势。

有可能一个品种一年以内，也就只有一到两波大的趋势。

假设你选择的交易标的比较少，很可能这些标的在一年或一段时间以内都没有一波大的趋势。

那么你的净值或资金，就有可能在震荡行情中不断亏损。

所以，一般 CTA 策略会做多品种分散。

它会同时交易十几个甚至几十个品种，确保在任意一个时间段，总有几个品种处在趋势当中。

通过趋势品种形成的盈利，来覆盖其余品种在震荡行情中带来的资金损失。

##### 与股票类或债券类策略相关性较低

CTA 策略本质上做的是波动和趋势。

所以，它的收益来源与股票、债券市场可能是完全不同的逻辑。

当股市暴跌的时候，CTA 策略往往可能因为做空了股指期货，而获得正收益。

#### 2. CTA 策略的局限性

##### 震荡行情中容易反复亏损

在长期震荡式横盘当中，策略容易被反复打脸。

##### 市场风格突变可能导致策略失效

CTA 策略大部分时候是通过历史数据回测产生的，比如策略中的一些参数。

当市场风格突变时，很可能导致策略参数或者策略本身失效。

##### 自带高波动性

期货自带高杠杆。

所以，CTA 策略的收益曲线通常比股票类策略更加陡峭，波动也会更大。

### 四、宏观择时策略

宏观择时策略是一种基于宏观指标以及政策变化，来主动调整大类资产配置仓位的投资策略。

它是一种自上而下的策略。

它首先判断当前经济环境处于什么周期，然后再决定在该周期、该阶段多配哪些资产，以及少配哪些资产。

#### 1. 美林时钟

比较常见的宏观择时周期框架是美林时钟。

美林时钟将 CPI 以及 GDP 作为横轴和纵轴，划分为四个象限。

这四个象限分别是：

1. 经济过热
2. 经济复苏
3. 滞胀
4. 衰退

在这四个不同象限里面，表现好的资产也不同。

比如在经济复苏时期，股票的表现就比较好。

该阶段的特点是：

1. GDP 稳健增长
2. CPI 比较稳定
3. PMI 扩张

而在衰退时期，通常表现为：

1. GDP 负增长
2. CPI 下降
3. PMI 收缩

这个阶段推荐增配的资产是债券和现金。

#### 2. 宏观择时策略的三步法

宏观择时策略的三步法是：

1. 根据宏观指标和政策进行周期识别
2. 在对应周期进行相应的资产映射
3. 根据配置好的资产，定期进行动态再平衡，也就是仓位调整

#### 3. 宏观择时策略的优势

宏观择时策略的优势在于抓大放小。

它忽略个股波动，而把握大的资产轮动机会。

有一句话叫做选择大于努力。

当你处在牛市的股票市场行情里，哪怕选择指数，都能够获得比较大的收益。

而如果处在熊市里，哪怕选股能力特别好，也很可能遭遇资产亏损。

#### 4. 宏观择时策略的劣势

它的主要劣势在于信号比较滞后。

因为宏观择时策略主要关注经济指标以及政策，比较少关注量价当前的具体位置。

所以，它的开仓成本可能并不如其他量价指标类策略那么优。

### 五、事件驱动策略

事件驱动策略只关注即将发生或正在发生的某个具体事件。

它通过预测该事件对未来资产价格的影响来获利。

具体来说，它会预测未来某一个时间点发生的事件，比如股票并购、业绩预告等等。

在事件发生之前，它会预测该事件对未来股价的影响。

如果认为这是一个利好事件，就提前买入。

如果认为这是一个利空事件，就提前卖出。

当事件真正发生、价格被重新定价时，策略获取相应收益。

#### 1. 事件驱动策略的核心原理

事件驱动策略的核心原理，是博弈不确定性的消除。

具体来说，就是赌事件落地之后，价格会被重新定价，也就是经常说的 Price in。

这也是我们经常说的：

我们交易的其实并不是某一个事件，而是大家对该事件的预期。

#### 2. 事件驱动策略的特点

事件驱动策略一般来说净值曲线比较平稳。

理论上，无论牛熊，只要事件成功，它就能够赚到钱。

并且因为持仓周期比较短，所以亏损理论上比较有限。

#### 3. 事件驱动策略的挑战

##### 信息不对称

一般来说，只有机构才拥有更强的尽调能力。

散户要做事件驱动，相对来说比较困难。

##### 黑天鹅事件

如果预期中的事件没有如期发生，就有可能导致事件驱动失败。

##### 流动性风险

事件驱动往往集中在少数股票或少数标的上。

一旦预期落空，或者开仓、平仓节奏没有把握好，很可能导致出货比较困难。

### 六、第二象限：波动小且择时的策略

第二象限是择时且波动小的策略。

这类策略不追求特别大的收益，而是追求净值尽可能平稳。

它就像 DOTA 里面的树精卫士：

虽然也需要在团战时进行切入和开大，但大部分时候，它的职责是加血、加护甲，保证血量平稳。

这一类策略主要包括：

1. 大类资产配置策略
2. 固收加策略

### 七、大类资产配置策略

大类资产配置策略和宏观择时策略非常像，但是它们追求的目标完全不一样。

宏观择时是根据经济周期或经济指标来配置资产，以获取更高收益。

而大类资产配置虽然也是配置多资产，但它追求的目标是利用不同资产之间的相关性来分散风险。

#### 1. 与宏观择时的区别

假设当前市场环境处于复苏期。

宏观择时策略可能会将大部分资产，比如 80%、90%，甚至全仓配置到股票。

而大类资产配置策略可能最多只配置 60% 或 70% 的股票。

剩下的仓位，依然会配置到债券或其他资产。

甚至有一些大类资产配置策略，只关注投资人的风险偏好，并根据风险偏好做一个类似固定比例的配比。

#### 2. 不同风险偏好的配置示例

如果一个投资者临近退休，或者是极度风险厌恶的投资者，那么可能采用保守型配置。

比如：

1. 20% 配置股票
2. 70% 配置债券

如果是风险偏好比较强的投资者，比例可能更激进一点。

比如：

1. 70% 配置股票
2. 20% 配置债券

#### 3. 大类资产配置的两个步骤

一般来说，大类资产配置有两个步骤：

1. 战略性配置
2. 战术性调整

战略性配置，是根据风险偏好设定一个比较大的、确定性的比例。

然后基于这个大比例，再进行一些战术性的调整。

#### 4. 大类资产配置策略的优势

大类资产配置策略的优势在于：

1. 风控做得比较好
2. 从长期来看能够穿越周期

#### 5. 大类资产配置策略的局限

它的局限也比较明显。

在单边牛市当中，它可能带来的收益会远远不如那些重仓买入股票的策略。

### 八、固收加策略

固收加策略是一种用债券打底，用权益资产，比如股票，进行增强的稳健型投资策略。

它追求的是稳稳的幸福。

它用 70% 甚至 90% 的债券作为底仓，而用不到三成的仓位做进攻，来增强收益。

#### 1. 固收加策略的典型特征

##### 股债搭配

对于大部分市场行情来说，债券和股票之间有一个跷跷板关系。

也就是股票涨，债券就跌。

所以进行股债搭配，就有一个东边不亮西边亮的平滑效果。

##### 绝对收益导向

固收加策略不追求相较于基准指数有多少收益增厚，也不追求相较于其他策略排名有多高。

它追求的是每年能够带来比较稳定的正收益。

比如年化 4% 到年化 6%。

##### 严格控制回撤

固收加策略会严格控制回撤。

所以，它特别适合那些风险厌恶型投资者。

#### 2. 固收加策略的风险点

##### 股债双杀

如果出现股债双杀，也就是股票跌，债券也跌，那么该策略也可能带来亏损。

##### 股票仓位过高

如果策略的股票仓位配置过高，可能会放大亏损。

##### 信用债违约

如果策略里面配置的信用债发生违约，也会对净值产生比较大的影响。

### 九、第三象限：波动小且非择时的策略

第三象限是波动小且非择时类的策略，也就是经常听到的中性策略。

这类策略包括：

1. 套利策略
2. 配对交易策略
3. 高频交易相关技术

此外，在一些套利类策略里面，高频交易技术是一种非常常见的手段。

这个象限的策略，就像 DOTA 里面的赏金猎人：

它并不追求在团战里面正面硬刚，而是依靠游走、标记，不断赚一些小钱，积少成多。

### 十、套利策略

#### 1. 套利策略的生活化例子

假设有一款同样的草莓，在山姆卖 15 块钱一斤，在盒马卖 16 块钱一斤。

这个时候，我们就可以在 15 块钱的市场里买入，并且在 16 块钱的市场里卖出。

赚到的这一块钱差价，就是套利利润。

#### 2. 套利策略的核心原理

这个例子揭示了套利的核心原理：

同一款商品或同一个资产，在扣除手续费以及其他调整之后，在任何时间、任何市场上的价格应该是一样的。

如果价格不一样，那么就有套利机会。

我们就可以低买高卖，赚取差价。

#### 3. 常见套利策略类型

##### ETF 套利

ETF 既有一级市场的申购和赎回，也就是基金净值，也有二级市场上的买卖价格。

当一级市场和二级市场上的价格不同，就可以进行套利。

##### 期现套利

期货里面有期现套利。

当期货和现货价格不同的时候，也会产生套利机会。

##### 跨期套利

同一个期货在不同交割月份之间，如果价差发生明显偏离，就会有套利机会。

##### 跨市场套利

同一个商品在不同交易所之间，如果存在价差偏离，也可能产生套利机会。

比如黄金在沪深交易所有黄金 ETF，在金交所有黄金实物以及黄金 TD。

同一个黄金在不同交易所出现不同价格，就有价差套利机会。

#### 4. 套利策略的特征

##### 风险较低

套利策略并不依赖于市场涨跌，而是依赖于价差回归。

当它开一个多头敞口时，一般会反向再开一个空头敞口，不会有单边敞口的暴露。

##### 与高频交易绑定紧密

套利机会转瞬即逝。

如果要吃到别人吃不到的套利机会，无论是交易系统、硬件，还是网络，都需要做很强的优化。

这些一般被称为高频交易。

##### 容量有限

一般来说，流动性不强的品种，套利机会比较大。

而流动性好的品种，套利机会比较小。

这就产生了一个悖论：

套利机会大的品种，流动性不好，因此容量小，能够承载的资金量比较小。

而流动性好、能够承载资金量大的品种，套利机会又比较少。

##### 单次利润很少

套利策略的单次利润非常少。

它是依靠高胜率以及反复交易，来获取持续稳定的收益。

### 十一、配对交易策略

配对交易策略和套利策略非常类似。

不过在这里，配对交易更多泛指统计套利。

所谓统计套利，寻找的可能不只是两个同样品种在不同市场以及不同交割月份之间的套利机会。

它还可以寻找相关性比较强的两个品种之间的套利机会。

比如：

1. 螺纹钢和铁矿石
2. 黄金和白银
3. 铝和锌
4. 5 年期国债和 10 年期国债

这些品种虽然不是同一个品种，但是它们之间存在中长期稳定的相关关系。

所以，当它们的价差或者价比出现比较明显的偏离时，就可以做它们的回归。

#### 1. 配对交易的操作逻辑

配对交易通常是：

1. 做多一个品种
2. 同时做空另一个品种
3. 赌价差会回归历史均值

#### 2. 配对交易的风险

配对交易和套利策略非常相似，但是它的风险相较于套利策略会稍微大一点。

因为它毕竟不是两个完全相同的品种。

很多时候，做配对交易回归时，并不知道组合需要拿多久，甚至需要扛很长时间的亏损。

#### 3. 配对交易需要特别注意的问题

做配对交易时，非常需要注意一点：

当两个品种之间的价格驱动逻辑发生根本性改变时，它们的价差中枢可能也会发生根本性改变。

这个时候，如果再用过去的价格做回归，很可能在未来很长时间内都难以回去。

这就会造成这笔配对交易的实际损失。

### 十二、高频交易

高频交易并不是一种投资策略，而更是一种技术实现方式。

一般来说，高频交易是用 C++ 实现的，当然也有少部分使用 Rust 实现。

#### 1. 高频交易的特点

##### 交易速度极快

高频交易的交易速度极快，持仓时间非常短。

通常是在毫秒、微秒甚至纳秒级别。

##### 单笔利润很少

高频交易的单笔利润非常少。

它依靠海量高胜率交易来累积收益。

##### 通常不隔夜持仓

高频交易一般不会有隔夜持仓。

##### 技术成本比较高

一般来说，高频交易首先需要将服务器托管到交易所机房。

同时，还要做很多高性能服务器配置，以及网络和代码方面的优化。

#### 2. 高频交易的常见应用场景

高频交易一般会用于以下场景：

1. 股票日内 T0 算法
2. 基于 L2 Tick 数据进行的订单簿分析
3. 做市策略
4. 套利策略
5. 股票打板策略

### 十三、第四象限：波动大且非择时的策略

第四象限是波动大且非择时类的策略。

这个象限最有代表性的策略，是股票多因子策略。

它就像 DOTA 里面的英雄卡尔：

招式非常多，技能非常多，连招也特别多。

但是，技能切换和释放相对来说比较复杂。

如果招式技能释放顺序比较好，就能产生比较大的伤害以及收益。

而如果技能释放拉垮、不连贯或者不合适，反而可能坑队友。

这一类策略主要包括：

1. 股票多因子策略
2. 指数增强策略
3. 轮动策略

### 十四、多因子策略

我们可以通俗地把因子理解为股票的某一个特征或特点。

比如：

1. 某个股票波动很大，可以理解为动量因子比较强
2. 某个股票估值比较便宜，可以理解为估值因子比较强
3. 某个股票市值特别小，可以理解为有小市值因子的特点

多因子就是将不同因子进行筛选和组合。

它不仅仅以单一维度进行选股，而是将这些因子筛选、组合、排序之后，构造出一个综合指标来进行股票筛选。

#### 1. 常见因子类型

##### 价值因子

比如市盈率、市净率。

核心逻辑是买便宜的股票。

##### 成长因子

比如营收增长率、净利润增长率。

它赚的是企业高增长的钱，也就是买企业更好的未来。

##### 动量因子

比如某个股票过去 N 个月、N 周的涨幅。

它赚的是趋势延续的钱，也就是通常所说的追涨杀跌。

##### 质量因子

比如 ROE、杠杆率。

##### 波动率因子

比如历史波动率。

#### 2. 构建多因子策略的步骤

一般构建一个多因子策略，有四个步骤。

##### 第一步：因子选取

因子选取的前提，是手头有一个因子池。

比如十几个，甚至几百、几千个质量好且相关性较低的有效因子。

这里特别需要注意，因子的相关性要低。

如果这些因子相关性很高，看似选了很多个不同因子，实际上它们底层逻辑都是一样的。

本质上，还是只选了一个因子。

##### 第二步：权重配置

当因子选取完之后，第二步要做的是权重配置。

也就是决定每个因子的话语权。

一般来说，等权重模型比较通用，使用也比较多。

##### 第三步：股票筛选与排序

配置完权重之后，根据这些因子的权重组合进行股票筛选。

筛选完之后，每个股票都会基于这些因子组合得到一个打分。

再基于这些打分进行排序，得到综合得分。

##### 第四步：买入与调仓

根据综合得分进行买入。

比如买入前 10% 或前 50 只股票。

买入之后，再进行定期调仓。

比如每个月调一次仓。

#### 3. 多因子策略的优势

##### 风险比较分散

单一因子可能阶段性失效。

但如果因子比较多，就可以确保在不同市场环境下，总有几个因子能够产生收益。

##### 市场容量特别大

在 A 股市场里面，很多量化策略都是股票多因子策略。

#### 4. 多因子策略的挑战

##### 因子失效

过去有效的因子，并不代表未来一直有效。

比如在 A 股市场里面，大部分时候小市值因子可能是有效的。

但是小市值因子也会发生定期回调和踩踏。

##### 过拟合

过拟合指的是，在历史上挖掘出表现看似特别好的因子，但实际上拟合的是噪音。

### 十五、指数增强策略

指数增强策略顾名思义，有两个关键词：

1. 指数
2. 增强

所谓指数，是指它会跟踪某一个指数。

比如：

1. 沪深 300
2. 中证 500
3. 中证 1000

它的基础目标，是至少要达到指数的贝塔收益。

第二个目标，也就是增强目标，是利用一些主动管理手段，在指数贝塔收益基础上，再创造一些阿尔法收益。

也就是比指数再多赚一点点。

#### 1. 指数增强策略的例子

假设某个指数增强策略跟踪的是沪深 300 指数。

在一定期限以内，比如最近一年，沪深 300 涨了 10%。

那么这个指数增强策略至少也得涨 10%。

这个赚的是贝塔收益，也就是市场整体平均收益。

在此之外，它还要做一个增强收益。

比如通过多因子选股以及择时等其他手段，在 10% 的基础上，再额外创造 2% 的收益。

也就是说，沪深 300 涨了 10%，但指数增强策略涨了 12%。

那么多出来的 2%，就是阿尔法，也就是增强部分。

#### 2. 指数增强的常见手段

##### 多因子选股

对于普通被动指数基金来说，它可能会进行沪深 300 的全样本复制。

但对于指数增强策略来说，它只需要复制一些关键特征类型的股票，保证整体波动和指数波动大致相当。

在复制基础之外，可以通过多因子选股的方式，给某些未来上涨潜力更大的股票更高配置权重。

如果配置正确，这些未来更有潜力的股票就会带来更多收益。

##### 打新增强

除了多因子选股之外，还可以通过打新方式进行收益增强。

##### 有限度择时

此外，还可以通过有限度择时进行增强。

比如基金经理预期未来指数可能发生比较明显的趋势性下跌，就可以提前减少一些持仓，或者调整一些仓位。

这样可以让指数增强策略的收益更加平稳，也更高一些。

#### 3. 指数增强策略的优势

指数增强策略的优势之一，是风格不漂移。

因为它有指数约束，所以投资者买指数增强策略时，大致能知道这个策略底层是怎样的资产标的。

#### 4. 指数增强策略的风险

##### 增强失效

有可能多因子选股选错了。

被选中的股票未来不仅没有上涨，反而可能下跌。

##### 规模不经济

指数增强策略规模增大之后，阿尔法会被摊薄。

换句话说，当同一个指数增强策略做的人越多，能够做增强的部分就会越少。

### 十六、轮动策略

轮动策略说白了，就是在一篮子资产里面不断切换。

它会不断卖出近期涨得比较多的资产，买入那些近期比较弱势，但你认为未来可能会有比较大上涨的资产。

#### 1. 常见轮动维度

##### 大类资产轮动

比如基于美林时钟，或者国内的货币利率模型，在股票、债券、商品之间进行轮动。

##### 行业轮动

在股票内部，可以根据产业政策以及行业景气度拐点，在不同类型的行业股票指数之间进行轮动。

##### 风格轮动

比如成长股和价值股之间的轮动。

##### 市值轮动

比如大盘股和小盘股之间的轮动。

#### 2. 轮动策略需要注意的前提

在讨论轮动策略时，特别需要注意一点：

轮动策略的前提条件是，轮动的资产或者策略，在未来至少是震荡行情，或者是牛市行情。

只有这样，轮动策略才能保证带来正的绝对收益。

如果轮动的资产在未来是下跌行情，或者处于熊市行情，那么轮动并不能避免整体策略亏损。

它只能减少策略损失。

### 十七、总结

按照波动大小以及择时或非择时两个维度，量化策略可以划分为四个象限。

第一象限是波动大且择时的策略，典型代表包括 CTA 策略、宏观择时策略和事件驱动策略。

第二象限是波动小且择时的策略，典型代表包括大类资产配置策略和固收加策略。

第三象限是波动小且非择时的策略，典型代表包括套利策略、配对交易策略，以及高频交易相关技术手段。

第四象限是波动大且非择时的策略，典型代表包括股票多因子策略、指数增强策略和轮动策略。

不同策略的收益来源、波动特征、风险点和适用场景都不一样。

理解这些策略之间的差异，有助于在量化投资中选择更适合自身目标和风险偏好的策略类型。


## 量化私募的两种模式

> 来源：用户上传文本 :contentReference[oaicite:0]{index=0}

### 一、国内主流量化私募的两种模式

国内主流的量化私募模式主要有两种：

1. 因子工厂模式
2. PM 团队模式

### 二、因子工厂模式

因子工厂模式是主流头部量化私募常见的一种玩法。

之所以称为“量化因子工厂”，是因为量化交易中的因子就像从流水线工厂中生产出来的一样，每一个环节都有明确分工。

在这种模式下，团队内部通常会有不同角色分别负责不同环节：

1. 有人专门负责挖掘因子
2. 有人专门利用因子构建模型
3. 有人专门负责编写代码，将策略模型落地为交易系统
4. 有人专门负责风控和交易管理

在这套流水线式的因子工厂模式下，量化并不是靠某一个人拍脑袋想出来的，而是依托数据和算法，批量生成、回测和迭代因子。

因子工厂模式的核心，是依靠数据和算力，不断生产和优化出能够赚钱的因子，然后再将大量好的因子打包组合成一个好的策略。

比如像幻方、九坤、明汯这些千亿级私募，基本上都是这种模式。

#### 1. 因子工厂模式的优点

##### 抗风险能力强

因子工厂模式的因子足够多，策略足够分散，因此不怕单一市场风格切换。

##### 可复制性强

这种模式不依靠明星基金经理，而是依靠整体团队形成比较稳定和持续的输出。

##### 规模上限高

因子工厂模式能够支撑百亿、千亿级别资金的运作。

#### 2. 因子工厂模式的缺点

##### 投入巨大

因子工厂模式需要自建算力、数据和团队，因此投入很大，烧钱也比较快。

##### 容易出现策略同质化

大家挖掘的可能都是非常相似的因子，这样就容易出现因子拥挤和踩踏。

##### 容易产生黑箱风险

因为因子太多，在将这些因子组合之后，有时候很难说清楚具体是哪一部分因子在赚钱。

### 三、PM 团队模式

与因子工厂模式相对的，就是 PM 团队模式。

这也是一些传统和中小型量化私募的主流做法。

简单来说，PM 团队模式就是投资经理带着自己的小团队，自己负责一条或几条策略，从研究到交易一把抓。

在这种模式下，团队高度依赖基金经理个人的能力和经验。

比如某些基金经理可能擅长做价值因子，或者擅长挖掘某一类市场异象因子。

这种模式比较依靠投资经理的个人经验和判断，团队都是围绕投资经理的策略思路来做研究和执行。

这非常类似于一个外科手术团队：由主刀医生负责核心判断，麻醉师以及其他角色辅助主刀医生完成这一场手术。

很多早期量化私募或者主观私募，经常使用这种模式。

#### 1. PM 团队模式的优点

##### 决策比较灵活

投资经理可以快速调整策略，以适应市场变化。

##### 成本比较低

PM 团队模式并不需要构建大规模算力和数据团队，整体比较小而美。

##### 特色鲜明

这种模式非常容易做出差异化策略，避开头部机构的一些竞争。

#### 2. PM 团队模式的缺点

##### 容易受到规模瓶颈限制

因为单一基金经理的策略容量有限，很难把规模做得非常大。

##### 人员依赖较强

如果核心投资经理离职，很可能导致该量化私募的某些策略失效，或者策略下线。

### 四、两种模式的比较

- 比较维度：驱动力
  - 因子工厂模式：依靠数据、算法和算力进行系统性驱动
  - PM 团队模式：依靠个人经验和人才驱动

- 比较维度：组织架构
  - 因子工厂模式：分工明确，各环节独立分工、相互配合
  - PM 团队模式：每个投资经理带着自己的小团队，全流程、全链路负责

- 比较维度：策略稳定性
  - 因子工厂模式：稳定性相对较高，通过大量因子分散风险，单一因子失效的风险较小
  - PM 团队模式：稳定性较低，高度依赖 PM 的状态，策略失效时较难快速调整

- 比较维度：规模
  - 因子工厂模式：可以支持百亿、千亿级别规模
  - PM 团队模式：规模相对有限

- 比较维度：迭代速度
  - 因子工厂模式：标准化流水线生产，策略和因子的产出、迭代速度较快
  - PM 团队模式：因子和策略的产出、构建相对较慢

- 比较维度：风险点
  - 因子工厂模式：策略容易同质化，造成因子拥挤
  - PM 团队模式：投资经理可能离职，带走核心策略和团队；个人交易风格也可能发生漂移；黑箱操作会带来不确定风险


### 五、未来行业发展趋势

从未来行业发展趋势来看，头部量化私募都在往因子工厂方向靠。

#### 1. 市场竞争越来越激烈

一方面，市场越来越卷，A 股散户占比在下降，机构占比在上升，简单因子也越来越容易失效。

因此，必须依靠算力和 AI，批量挖掘一些新的另类因子。

#### 2. 资金规模越来越大

市场资金规模越来越大，百亿、千亿级别的资金规模，靠单个 PM 很难管理过来。

因此，必须依靠系统和流水线的方式来分散风险、提升效率。

#### 3. 人才和成本门槛越来越高

随着 AI、低延时等技术的发展，一般只有工厂化的大型团队，才能够将这些新技术有效利用起来。


## 交易的部署模式

### 一、话题引入

各位小伙伴们大家好。

今天我们来探讨一个在股票交易中比较实际，但容易被忽略的话题：交易的物理距离。

所谓物理距离，就是当你从点击下单，到该指令抵达交易所服务器这一段物理传输的距离。

根据部署模式的不同，总共有三类方式：

1. 本地方式
2. 单节点托管
3. 两地分仓

### 二、本地方式

本地方式是最常见的个人交易方式。

投资者在本地通过互联网交易客户端，来交易沪深两个交易所的股票。

由于物理距离的影响，你离哪个交易所近，那么可能发到哪个交易所就快一些。

这种方式难以避免互联网传输带来的延时，是速度最慢的一种方式。

### 三、单节点托管

第二种方式是单节点托管。

所谓单节点托管，就是你将自己的交易服务器托管到某个交易所的机房。

比如，你可以托管到深圳的机房，那么交易深圳的股票就有很大的速度优势。

但是，交易上海交易所的股票，还是会有来回的网络延时。

### 四、两地分仓

第三种最快的方式是两地分仓，也叫一户两地。

这种方式适用于高频交易。

具体来说，你会在上海和深圳的交易所机房都部署一台交易服务器：

1. 上海的股票，就在上海报盘
2. 深圳的股票，就在深圳报盘

所以，报单延时非常低。

同时，资金会根据需求进行实时转账。

因此，这种方式的交易效率非常高。

### 五、三种方式回顾

#### 1. 本地部署

本地部署是大多数散户的选择。

交易速度受限于物理距离和普通互联网的延时。

#### 2. 单节点托管

单节点托管，是将交易服务器搬到其中一个交易所的家门口。

这会让交易该交易所的股票有一个速度上的质的飞跃。

但是，跨交易所交易仍然会有延时。

#### 3. 两地分仓

两地分仓属于顶配方案。

沪深交易所的机房都有自己的服务器。

这就意味着，无论是交易沪市还是深市，都可以就近起跑。

## 如何用期权合成期货

### 一、主题引入

各位小伙伴们大家好。

本期视频我们通俗地解释一下：如何用期权合成一个期货。

### 二、现货与期货的风险特征

#### 1. 现货的风险特征

我们先来看一个普通现货的风险特征。

在图中：

1. 横轴代表现货价格
2. 纵轴代表你的盈亏

假如你在某一时刻买入了该资产标的，那么随着未来价格上涨，你的盈利会不断增加。

#### 2. 期货的风险特征

期货的风险特征和现货相比，首先它的价格变成了未来某一时刻的到期价格。

而期货一般来说既可以做多，也可以做空。

但无论是做多还是做空，它的风险特征依然是线性关系。

### 三、如何用期权合成期货

如何用期权来合成一个期货合约？

先上结论。

#### 1. 合成期货多头

要合成期货的多头，需要：

1. 买入一个看涨期权
2. 卖出一个看跌期权

#### 2. 合成期货空头

要合成期货的空头，需要：

1. 买入一个看跌期权
2. 卖出一个看涨期权

#### 3. 注意事项

这里的看涨期权和看跌期权，需要满足两个条件：

1. 行权价相同
2. 到期日相同

### 四、用行情软件演示

我们先用行情软件演示一下该公式是否正确。

打开 Wind，随便选择一个期权。

例如选择 11 月到期的期权，行权价都是 3100 元。

#### 1. 合成期货多头

合成期货多头，需要：

1. 买入一个看涨期权
2. 同时卖出一个相同行权价的看跌期权

此时得到的线性曲线，就是合成期货多头的风险特征。

合成的期货多头，会随着价格不断上涨，盈利也不断增加。

#### 2. 合成期货空头

合成期货空头，需要：

1. 买入一个看跌期权
2. 同时卖出一个看涨期权

此时，随着价格不断上涨，亏损不断增加的曲线，就是合成期货空头的风险特征曲线。

### 五、合成期货多头举例

下面进一步举例说明合成期货多头和合成期货空头。

假设有这样一个资产标的：

1. 股票当前价格是 100 元
2. 行权价是 100 元
3. 三个月以后到期
4. 买入看涨期权的权利金是 5 元
5. 看跌期权的权利金是 4 元

如果你要合成一个期货多头，当股票到期价格是 80 元时：

1. 看涨期权不行权
2. 买入看涨期权亏损 5 元权利金
3. 卖出的看跌期权被行权
4. 看跌期权亏损 20 元
5. 总损益是亏损 25 元

当到期价格是 120 元时：

1. 买入的看涨期权会被行权
2. 看涨期权赚 15 元
3. 卖出的看跌期权不会被行权
4. 看跌期权赚 4 元权利金
5. 总损益是盈利 19 元

所以，合成的期货多头，它的盈亏曲线就是一条向上的线性曲线。

### 六、合成期货空头举例

同理，当你合成一个期货空头时：

1. 卖出一个看涨期权
2. 买入一个看跌期权

当到期价格是 80 元时：

1. 卖出的看涨期权不会被行权
2. 赚取 5 元权利金
3. 买入的看跌期权行权之后赚 17 元
4. 总损益是盈利 22 元

当到期价格是 150 元时：

1. 卖出的看涨期权被行权
2. 看涨期权亏损 45 元
3. 买入的看跌期权不会被行权
4. 亏损 3 元权利金
5. 总亏损是 48 元

所以，这根向下的曲线，就是合成期货空头的风险特征曲线。

### 七、合成期货与直接交易期货的对比

最后，我们将合成期货和直接交易期货做一个简单对比。

#### 1. 成本

从成本上来看，用期权合成的期货，成本是权利金的净额，约 1% 到 2%。

而直接交易期货，成本是保证金，约 12% 到 15%。

#### 2. 风险特征

从风险特征上来看，用期权合成的期货与直接交易期货，二者都是线性的风险特征。

#### 3. 灵活性

如果直接交易期货，一般来说期货有固定的到期日，以及相应的一些交割条款。

而用期权合成期货，可以灵活调整行权价，并选择相应的到期日。

#### 4. 资金效率

从资金效率上来看，用期权合成期货的资金效率相对较高。

它相当于只需要 1% 或 2% 的保证金。

而直接交易期货，保证金相对来说会更多一点，也就是杠杆会低一点。

反过来说，杠杆越高，代表风险也会越大。

#### 5. 操作复杂度

从操作复杂度上来看，期权合成期货的复杂度较高。

因为你需要同时管理两个期权。

而直接交易期货相对来说比较简单，只需要买卖标准化的现成期货合约。

### 八、总结

用期权合成期货，核心方法是：

1. 合成期货多头：买入看涨期权，卖出看跌期权
2. 合成期货空头：买入看跌期权，卖出看涨期权

同时需要注意，看涨期权和看跌期权的行权价与到期日必须相同。

期权合成期货在资金效率和灵活性上有一定优势，但操作复杂度更高，杠杆也可能更高，因此风险也需要特别关注。

## 通俗理解期权平价公式

### 一、主题引入

各位小伙伴们大家好。

今天我们通俗地来理解一下期权平价公式。

### 二、四种基础资产

首先，我们来看假设存在的四种资产。

#### 1. 看涨期权

所谓看涨期权，就是约定在未来某一个时间点或时间段，能够以某一个价格买入某种标的的一种权利。

我们假设这个看涨期权在一年后，能够以 100 元的行权价买入某种股票。

#### 2. 看跌期权

看跌期权和看涨期权类似。

它约定在未来某一个时间点或时间段，能够以某一个价格卖掉某一个资产。

我们假设该看跌期权在一年后的行权价是 100 元，也就是能够以 100 元的价格卖掉某只股票。

#### 3. 正股

第三个资产，是这两个期权对应的正股。

#### 4. 一年后到期的债券

第四个资产，是一年后到期的债券。

该债券在当前的价格为 PV，也就是该债券当前的现值。

### 三、两种组合

根据这四种资产，假设现在有两种组合。

#### 1. 组合一：S + P

第一种组合是：

S + P

也就是以 S 的价格买入一个股票，同时以 P 的价格买入一份看跌期权。

#### 2. 组合二：C + PV

第二种组合是：

C + PV

也就是以 C 的价格买入一份看涨期权，同时以 PV 的价格买入一份一年后到期的债券。

我们假设该债券在一年后的终值是 100 元。

### 四、左边组合：S + P

我们先来看左边的组合：

S + P

其中：

1. S 是正股
2. P 是看跌期权

它本质上就是买了一份股票，再加了一份保险。

这里把看跌期权理解为保险，意思是：

假设股票在一年以后价格跌了非常多，但是你依然能够以 100 元的行权价格将这只股票卖出去。

所以，看跌期权实际上就是一份保险。

#### 1. 情形一：股票上涨

假设股票上涨，比如股票未来价格涨到了 120 元。

这个时候，保险作废。

因为当股票上涨到 120 元时，你并不会以 100 元的价格把它卖掉。

此时，你的收益是：

股票相较于你的成本价涨多少，你就赚多少。

#### 2. 情形二：股票下跌

假设股票价格下跌，并且跌破了 100 元，比如跌到了 80 元。

这个时候，不管股票价格跌多少，你都有一个看跌期权，可以以 100 元的保底价格将它卖掉。

所以，第二种情形下，不管股票跌多少，你都有 100 元作为保底。

### 五、右边组合：C + PV

我们再来看右边的组合：

C + PV

也就是看涨期权加上一份债券。

它本质上是债券加上一个保险。

同样，我们来看两种情形。

#### 1. 情形一：股票上涨

当股票价格上涨时，看涨期权生效。

因为无论股票价格涨到多贵，你总是能够以 100 元的价格买入该股票。

所以，股票涨多少，你就赚多少。

#### 2. 情形二：股票下跌

当股票价格下跌时，比如股票跌破了 100 元。

这个时候，你并不会以 100 元的行权价买入股票。

所以，这份看涨期权就作废。

但是，你有一份一年后到期为 100 元的债券作为保底。

所以，这个时候你依然有 100 元的保底价。

### 六、两个组合的收益结构比较

现在我们比较一下左边组合和右边组合。

当股票上涨和股票下跌时，这两种组合的收益结构都是完全一致的。

具体来说：

1. 当股票上涨时，它们都是涨多少赚多少
2. 当股票下跌时，它们都有 100 元作为保底

所以，收益结构相等的两个组合，它们的定价理应相同。

如果它们的定价不同，就存在套利机会。

如果存在套利机会，那么就可以抓住该套利机会，从而产生收益。

### 七、期权平价公式

这个时候，我们再回顾一下该公式：

S + P = C + PV

左边的组合等于右边的组合。

这个式子就叫做期权平价公式。


## 算法交易：TWAP、VWAP 与 POV

### 一、算法交易的引入

小伙伴们大家好。

今天我们介绍一下算法交易。

我们先假设一个场景：现在你有 10 万手股票需要买入。

如果将这 10 万手股票集中买入，就会造成市场价格上涨，从而导致买入成本变差，这就是冲击成本的上升。

如果分批量买入，又有可能造成这 10 万手订单不能够完全被执行。

所以，如何能让大额订单在成交的同时，尽可能控制住成本，是一个需要考虑的问题。

而解决这个问题的方法，就是算法交易。

### 二、算法交易的应用情况

在 2024 年，华泰证券通过算法交易的金额已经达到了 4 万亿元，占 A 股总交易额的 1.56%。

此外，在公募基金前 50 名当中，有 85% 的公募基金已经采用了算法交易。

头部券商，比如中信、华创、中金，均有自己的自研算法交易系统。

### 三、算法交易的基本逻辑

算法交易的逻辑，是通过历史数据寻找一些统计规律，比如：

1. 成交时间
2. 成交量
3. 价格分布

然后再根据这些规律制定下单规则。

根据这套制定逻辑，算法交易或者算法下单可以分为几个主要类型。

### 四、常见算法交易类型

#### 1. TWAP

第一类常用算法叫做 TWAP。

T 是 Time，也就是时间的意思。

所以，TWAP 算法交易是根据时间规律制定交易规则。

#### 2. VWAP

与 TWAP 相对应的还有 VWAP。

V 是 Volume，也就是成交量的意思。

所以顾名思义，VWAP 是根据成交量规律来制定交易规则。

#### 3. POV 与 IS

基于 VWAP 的改进，又有 POV 算法和 IS 算法。

在目前国内算法交易市场中，TWAP 和 VWAP 占据了半壁江山。

而基于 VWAP 改进的 POV 算法，也占据了 36% 的比例。

剩下的 IS 算法，因为会让订单在短时间内集中成交，并可能引发市场剧烈波动，所以受到监管的严格限制。

因此，IS 算法在国内使用得并不是很多。

下面主要介绍主流的 TWAP、VWAP 和 POV 算法。

### 五、TWAP 算法

TWAP 是 Time Weighted Average Price 的简称，也就是时间加权平均价格。

这种算法的目标，是让交易最后的平均成本，尽量贴近时间加权价格。

#### 1. TWAP 的基本逻辑

TWAP 说白了，就是在给定的交易时间段内，根据时间间隔均匀地下单。

比如，假设我们需要在 9:30 到下午 2:10 之间把单子下完。

那么就可以将订单分割为 N 等份，然后在每个时间段内都下同等比例的单子。

又或者，我们希望将订单在 9:40 到 11:10 之间全部下完。

那么委托就需要在这段时间内被均匀分割为多个等份，并被均匀地下出去。

这就是 TWAP 算法的基本逻辑。

#### 2. TWAP 的适用场景

根据 TWAP 的基本逻辑可以发现，它适用于那些流动性较高，且无明显成交波动的市场。

因为如果成交量波动特别大，采用均匀下单的方式，可能在某一个时间段内无法将委托全部成交。

#### 3. TWAP 示例

假设某机构需要在三个小时内买入 10 万股某只股票。

如果采用传统的 TWAP 策略，就可以将这三个小时，也就是 180 分钟，均分为 18 个间隔。

也就是每 10 分钟执行一次。

每次下单量就是：

10 万股 ÷ 18 个间隔 = 5556 股

#### 4. 传统 TWAP 的问题

传统 TWAP 策略存在一个问题。

当市场流动性突然发生变化时，比如市场暴跌或暴涨，就可能导致这些委托无法完全成交。

所以，我们需要对传统 TWAP 策略做一些优化。

#### 5. TWAP 的动态调整

优化方式就是动态调整。

为了防止暴露均匀下单的交易意图，可以将固定的 10 分钟时间间隔，变成随机时间间隔。

比如可以将 10 分钟调整为动态的 10 到 15 分钟。

第一次可以在 8 分钟内下完，第二次可以在 12 分钟内下完。

同时，订单量也可以加入波动范围。

比如在订单量的基础上，上下增加 20% 的波动幅度。

此外，当市场剧烈波动时，下单策略也可以进行调整。

比如，当市场快速上涨时，如果仍然以均匀速度买入，后面的成交成本将会非常高。

所以在价格上涨时，可以自动缩短时间间隔，加速订单成交。

如果波动率发生变化，也需要让订单在短时间内成交掉。

还有一种情况是，当接近尾盘结束交易时，仍然有部分订单没有成交。

此时就需要在尾盘将未成交部分集中成交掉。

#### 6. TWAP 的代码逻辑

假设现在有 10 万股股票，需要在 300 秒内完成成交，并且每 30 秒下一次单。

那么它有三个参数：

1. 总共需要成交的股数
2. 需要成交的交易时段长度
3. 成交间隔

它的逻辑是：

如果当前时间和开始交易时间之间的差值小于 300 秒，就每隔 30 秒进行一次下单。

下单股数为：

10 万股 ÷（300 秒 ÷ 30 秒）

也就是分 10 次下单。

每次下单 1 万股。

这里如果代码里使用两个斜杠，表示取整，防止除出来是零点几股。

### 六、VWAP 算法

前面提到，TWAP 适合在成交量无明显波动的市场下进行交易。

当成交量波动比较大时，就可以使用 VWAP 算法。

#### 1. VWAP 的基本逻辑

VWAP 的简单逻辑是：

成交量大的时候多下几笔单，成交量小的时候交易量也跟着减小。

VWAP 算法会根据历史成交数据来预测成交量的多或少。

比如，历史成交量比较高的时候，委托就会多一点。

历史成交量比较小的时候，委托也相应小一点。

#### 2. VWAP 示例

假设根据某只股票的历史成交数据显示：

1. 开盘前 30 分钟成交量占全天的 20%
2. 盘中成交量占全天的 50%
3. 收盘阶段成交量占全天的 30%

假设我们有 10 万股股票需要执行。

那么就可以将这 10 万股股票，根据开盘、盘中和收盘的比例进行分配：

1. 开盘：2 万股
2. 盘中：5 万股
3. 尾盘：3 万股

然后再将这 2 万股、5 万股和 3 万股，根据时间间隔等量地下出去。

这就是一个 VWAP 算法的例子。

#### 3. VWAP 的动态调整

VWAP 算法也可以进行动态调整。

比如，当盘中的实际成交量高于预测成交量时，就需要在盘中时段加大报单量。

假设原定每笔报 1250 股，可以调整为每笔报 1500 股。

#### 4. VWAP 的代码逻辑

VWAP 算法代码中的一个核心变量，是不同时间段的下单比例。

然后将总下单量乘以该比例，就可以得到在该时间段内需要下单的数量。

#### 5. VWAP 的问题

VWAP 算法也存在一些问题。

它过度依赖历史成交量的预测。

如果历史成交量预测不准确，或者当前市场发生了很大变化，历史成交量对当前市场的参考意义不大，那么 VWAP 算法就可能失效。

为了解决这个问题，POV 算法就登场了。

### 七、POV 算法

POV 是 Percentage of Volume 的简称，也就是成交量百分比算法。

它是 VWAP 算法的一个改进。

POV 会根据市场实时成交量和一定比例，动态调整订单量。

#### 1. POV 与 VWAP 的区别

VWAP 参考的是历史成交量。

而 POV 参考的是实际成交量。

它会根据实时成交量来确定自己的下单比例。

#### 2. POV 示例

假设某机构需要卖出 10 万股股票，采用 POV 策略进行算法下单。

假设 POV 的参与率是 10%。

所谓 10%，意思是：

当市场成交量是 1000 股时，你的成交量就是 1000 股乘以 10%，也就是 100 股。

回到这个例子。

第一步，需要实时监控市场。

当发现当前一分钟市场成交量是 5000 股时，通过 POV 算法计算，得到下单量应该是：

5000 股 × 10% = 500 股

当下一分钟市场成交量突然飙升至 8000 股时，下一分钟的下单量就要调整为：

8000 股 × 10% = 800 股

#### 3. 盘口限制

POV 算法还会受到盘口限制。

设置盘口限制，是为了防止冲击成本过大。

比如要卖出时，对手盘只有 300 股买一价委托。

这个时候，如果要卖出 800 股，就可能导致价格成本飙升。

所以在盘口限制下，哪怕目标是卖出 800 股，但在当前买一盘口条件下，最多可能只成交 300 股的 20%，也就是 60 股。

所以最后的实际下单量，就是以下两个数之间的最小值：

1. 800 股
2. 300 股 × 20%

剩余没有成交的部分，则放到下一分钟处理。

#### 4. POV 中的成交量预测

这里可以思考一个问题：

POV 算法的下单量，是根据市场实时成交量乘以一个比例得到的。

但实时成交量，比如当前一分钟的成交量，在这一分钟还没有走完的时候，怎么知道这一分钟的成交量是多少？

这就涉及预测问题。

一般来说，有一个比较简单的预测逻辑：

通过已经发生的成交量和已经用去的时间，来估计一分钟内的全部成交量。

这就是一个简单的预测方法。

#### 5. POV 的代码逻辑

POV 的代码逻辑大致如下：

1. 获取当前时间段内的预测成交量
2. 用预测成交量乘以参与率，得到目标下单量
3. 计算当前盘口限制，得到最多能挂多少股
4. 在目标下单量、盘口限制和剩余订单量之间取最小值
5. 将这个最小值作为最后需要委托出去的报单量

#### 6. POV 的缺陷

POV 算法也有一些缺陷。

当市场成交量骤降时，POV 算法很可能导致委托无法全部执行完成。

### 八、总结

算法交易的核心，是在大额订单成交的同时，尽可能降低冲击成本，并提高执行效率。

常见算法包括：

1. TWAP：根据时间均匀下单，适合成交量较稳定的市场
2. VWAP：根据历史成交量分布下单，适合成交量有明显分布规律的市场
3. POV：根据实时成交量动态下单，是对 VWAP 的进一步改进

不同算法适用于不同市场环境，也各自有局限。

TWAP 简单稳定，但容易暴露交易意图，也不适合成交量波动较大的市场。

VWAP 更贴近成交量分布，但依赖历史成交量预测。

POV 能够跟随实时成交量变化，但在市场成交量骤降时，可能导致订单无法全部完成。
