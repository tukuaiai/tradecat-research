# 量化交易学习集合

---

## 1. 领域总览：这个领域到底在解决什么问题？

量化交易的核心问题是：**能否把一个关于市场的假设，转化成可验证、可复现、可执行、可风控的交易系统？**

它不是简单地“用机器赚钱”，而是同时处理四件事：

| 层次  | 核心问题                       |
| --- | -------------------------- |
| 研究层 | 市场里是否存在可解释、可重复、可交易的收益来源？   |
| 统计层 | 这个收益是否显著，还是数据挖掘、过拟合、幸存者偏差？ |
| 工程层 | 这个策略能否在真实数据、真实成本、真实延迟下运行？  |
| 风控层 | 它在极端行情、拥挤交易、流动性枯竭时会不会崩？    |

它重要，是因为现代金融市场已经高度电子化，交易、做市、组合管理、风险控制、执行算法、指数增强、CTA、套利、机器学习选股，都不同程度依赖量化系统。CMU 的市场微观结构与算法交易课程把这一领域覆盖为：算法交易、高频策略、最优执行、成交质量分析、限价订单市场与市场结构等主题。([卡内基梅隆大学][1])

相邻领域包括：资产定价、金融计量、统计学习、机器学习、市场微观结构、投资组合管理、软件工程、数据库、分布式系统、交易所规则、合规监管。

初学者最容易误解的是：
**把量化交易误认为“找一个神奇指标”或“训练一个模型预测涨跌”。** 专业量化更关心的是：数据是否干净、样本是否独立、成本是否真实、组合是否稳健、回撤是否可承受、交易是否可执行。

## 2. Philosophy：该领域的“哲学层”知识

### 主干知识树

```text
量化交易
├── A. 市场观
│   ├── 有效市场假说：价格已反映信息
│   ├── 行为金融：人类偏差造成可利用模式
│   ├── 风险溢价：收益是承担系统性风险的补偿
│   ├── 市场微观结构：价格由订单、流动性和制度生成
│   └── 交易拥挤：有效策略会被资本挤压
│
├── B. 模型观
│   ├── 统计套利：均值回复、协整、配对交易
│   ├── 趋势跟踪：动量、时间序列趋势、CTA
│   ├── 因子投资：价值、动量、质量、低波、规模、盈利、投资
│   ├── 机器学习：特征、标签、验证、泛化
│   └── 最优执行：冲击成本、VWAP、TWAP、订单拆分
│
├── C. 工程观
│   ├── 数据获取
│   ├── 清洗对齐
│   ├── 特征工程
│   ├── 回测框架
│   ├── 组合构建
│   ├── 风险控制
│   ├── 执行系统
│   └── 监控与复盘
│
├── D. 组织观
│   ├── 研究员
│   ├── 交易员
│   ├── 数据工程师
│   ├── 执行工程师
│   ├── 风控/合规
│   ├── 基金经理
│   ├── 券商/交易所
│   └── 数据供应商
│
└── E. 评价观
    ├── 样本外表现
    ├── 夏普率/信息比率
    ├── 最大回撤
    ├── 换手率
    ├── 容量
    ├── 成本敏感性
    ├── 稳健性
    └── 资金曲线可解释性
```

### 起源与发展动机

量化交易来自三个动机：
第一，用数学与统计替代纯主观判断；第二，在大量资产中系统发现规律；第三，把研究、组合、执行、风控工业化。

它的发展主轴大致是：

| 阶段          | 主轴                         |
| ----------- | -------------------------- |
| 1950s—1970s | 现代投资组合理论、CAPM、有效市场假说       |
| 1980s—1990s | 金融计量、衍生品定价、统计套利、因子模型       |
| 2000s       | 电子交易、订单簿、高频交易、执行算法         |
| 2010s       | 大数据、另类数据、机器学习、因子拥挤         |
| 2020s       | AI 辅助研究、自动化研发、监管强化、算力与数据治理 |

Fama-French 的 1992 年横截面收益研究和后来的五因子模型，是因子投资与量化选股的重要基础；AQR 对价值、动量、套利/ carry、防御/质量等因子的长期研究，则代表了现代机构化因子视角。([Wiley Online Library][2])

### 核心假设与基本共识

| 假设                    | 含义                   |
| --------------------- | -------------------- |
| 市场并非完全随机              | 但可利用信号很弱、会衰减         |
| 收益来自风险、行为偏差、结构摩擦或信息优势 | 不能只看收益，要解释来源         |
| 回测不是现实                | 成本、滑点、冲击、延迟、容量都会改变结果 |
| 统计显著不等于可交易            | 样本外、交易成本、风控后才算数      |
| 交易系统是生产系统             | 不是 notebook 里的一条收益曲线 |

### 主要流派与分歧

| 流派      | 核心观点              | 分歧点               |
| ------- | ----------------- | ----------------- |
| 有效市场派   | 超额收益很难持续          | 反对过度相信 alpha      |
| 行为金融派   | 人类偏差创造动量、反转、错误定价  | 是否能长期交易化          |
| 风险溢价派   | 收益是承担风险补偿         | alpha 与 beta 边界模糊 |
| 市场微观结构派 | 价格由交易机制、流动性、订单流形成 | 更重视执行与制度          |
| 机器学习派   | 非线性模型能提取弱信号       | 容易过拟合、解释困难        |
| 高频做市派   | 优势来自速度、报价、库存管理    | 门槛高，普通人不适合直接切入    |

Maureen O’Hara 对市场微观结构的定义很关键：它研究市场规则、设计和“管道”如何影响流动性与价格发现。([Cornell SC Johnson][3])

## 3. Technique：该领域的“技巧层”知识

### 必须掌握的核心术语

| 术语                | 必须理解的意思             |
| ----------------- | ------------------- |
| Alpha             | 扣除风险暴露与成本后的超额收益     |
| Beta              | 对市场或系统性因子的暴露        |
| Factor            | 能解释或预测资产收益的共同变量     |
| Signal            | 交易信号，通常是排序、预测值或持仓权重 |
| Backtest          | 用历史数据模拟策略           |
| Out-of-sample     | 样本外验证               |
| Look-ahead bias   | 使用了当时不可能知道的信息       |
| Survivorship bias | 只看幸存资产，忽略退市/失败样本    |
| Slippage          | 理论成交价与真实成交价差异       |
| Market impact     | 自己交易对价格造成的冲击        |
| Turnover          | 换手率                 |
| Capacity          | 策略可容纳资金规模           |
| Drawdown          | 回撤                  |
| Sharpe Ratio      | 单位波动收益              |
| Information Ratio | 相对基准的超额收益/跟踪误差      |
| VWAP/TWAP         | 成交量加权/时间加权执行算法      |
| Order book        | 买卖盘订单簿              |
| Fill rate         | 成交率                 |
| Kill switch       | 异常时停止交易的风控开关        |

### 入门者必须训练的技能

| 技能          | 训练方式                                 |
| ----------- | ------------------------------------ |
| Python 数据处理 | pandas / numpy / polars 处理行情、财务、因子数据 |
| 时间序列处理      | 对齐、滞后、滚动窗口、缺失值、复权                    |
| 金融统计        | 收益率、波动率、协方差、t 检验、回归、残差               |
| 回测设计        | 避免未来函数、处理成本、调仓、仓位、约束                 |
| 因子研究        | 分组回测、IC、Rank IC、分层收益、换手              |
| 组合构建        | 等权、风险平价、均值方差、约束优化                    |
| 风控          | 止损、仓位限制、敞口限制、回撤控制                    |
| 执行理解        | 限价单、市价单、撮合、滑点、流动性                    |
| 复盘能力        | 把收益分解为因子、行业、风格、交易成本和偶然性              |

| 类别    | 工具                                                                |
| ----- | ----------------------------------------------------------------- |
| 研究语言  | Python、R、SQL                                                      |
| 数据处理  | pandas、numpy、polars、duckdb                                        |

### 专业与业余的分界线

| 业余理解     | 专业理解                   |
| -------- | ---------------------- |
| 回测收益高就好  | 样本外、成本后、风险后、容量后才有意义    |
| 找指标      | 提出可解释假设                |
| 优化参数     | 控制过拟合                  |
| 只看年化收益   | 看收益来源、回撤、换手、容量、稳定性     |
| 用免费行情随便测 | 关注复权、退市、公告时间、交易日历、数据延迟 |
| 模型越复杂越好  | 简单、稳健、可解释、可监控更重要       |
| 只做预测     | 预测、组合、执行、风控缺一不可        |

## 4. Fact：该领域的“事实层”知识

### 必须掌握的核心事实

| 事实              | 为什么重要                   |
| --------------- | ----------------------- |
| 多数公开策略会衰减       | 市场会学习，资金会拥挤             |
| 交易成本决定很多策略生死    | 高频、高换手策略尤其敏感            |
| 数据质量比模型复杂度更重要   | 错误数据会制造虚假 alpha         |
| 回测容易骗人          | 未来函数、过拟合、幸存者偏差极常见       |
| 真实交易是系统工程       | 研究、数据、组合、执行、风控、监控都要闭环   |
| 因子收益不是稳定工资      | 因子会长期失效、反转、拥挤           |
| 高频交易不是普通学习者优先方向 | 需要低延迟基础设施、交易所连接、硬件与合规资源 |
| 监管是领域核心约束       | 程序化交易需要报告、监控、风控与异常处理    |

CRSP 官方说明其美国股票数据库覆盖活跃与非活跃证券，并强调 survivor-bias-free 历史数据对严谨研究和回测的重要性；SEC EDGAR 也提供公司申报和 XBRL 数据 API，适合作为基本面数据学习入口。([Center for Research in Security Prices][5])

### 代表性案例

| 类型   | 案例                       | 学什么                |
| ---- | ------------------------ | ------------------ |
| 因子研究 | Fama-French 三因子/五因子      | 资产收益可以被系统因子解释      |
| 趋势跟踪 | CTA / managed futures    | 时间序列动量与危机 alpha    |
| 统计套利 | 配对交易、协整篮子                | 相对价值、均值回复          |
| 执行算法 | VWAP/TWAP/POV            | 大单拆分与冲击成本          |
| 失败案例 | Knight Capital 2012      | 软件部署、风控开关、监控失败可能致命 |
| 市场事件 | 2010 Flash Crash         | 流动性、订单流、自动交易相互作用   |
| 数据基础 | Fama-French Data Library | 因子收益和研究复现入口        |

Knight Capital 2012 年事件中，SEC 宣布其因违反 Market Access Rule 支付 1200 万美元和解；这类案例说明，量化交易的风险不只是模型错，还包括部署、权限、监控和异常处置。([美国证券交易委员会][6])

### 值得了解的背景事实

| 背景      | 价值                 |
| ------- | ------------------ |
| 交易所撮合机制 | 理解限价单、市价单、队列位置     |
| 做市商模式   | 理解买卖价差、库存风险        |
| 基金行业结构  | 理解量化私募、对冲基金、公募指数增强 |
| 监管趋势    | 理解程序化交易不是自由实验场     |
| 另类数据    | 理解数据壁垒与合规风险        |

中国市场方面，上交所已发布《程序化交易管理实施细则》，自 2025 年 7 月 7 日施行，目标是加强程序化交易监管、维护交易秩序和市场公平。([华电国际][7])

### 可以暂时跳过的碎片事实

| 暂时不学             | 原因            |
| ---------------- | ------------- |
| 超低延迟 FPGA        | 对初学者投入产出比低    |
| 复杂期权做市 Greeks 系统 | 需要衍生品和交易系统基础  |
| 加密货币链上高频 MEV     | 过于细分，监管与结构变化快 |
| 深度强化学习自动交易       | 学术很热，实盘难度极高   |
| 每天追逐新因子论文        | 容易陷入碎片化学习     |

12. 数据是否存在幸存者偏差？
13. 能否解释每一笔交易的理由？

## 6. 直接学习“困难内容”的路径

初学者不要先沉迷“简单指标”，而应该尽早碰困难内容，因为困难内容决定你是否真正入门。

### 应该优先学习的困难内容

| 困难内容   | 为什么优先              |
| ------ | ------------------ |
| 金融计量   | 避免把噪声当规律           |
| 市场微观结构 | 理解为什么纸面收益无法成交      |
| 回测偏差   | 这是量化新手最大陷阱         |
| 交易成本模型 | 直接决定策略生死           |
| 因子暴露分解 | 判断 alpha 还是伪 alpha |
| 组合与风控  | 单个信号不等于可交易组合       |
| 样本外验证  | 区分研究与幻觉            |
| 执行算法   | 理解从信号到订单的损耗        |

### 具体学习顺序

```text
第 1 步：收益率、波动率、回撤、相关性
第 2 步：CAPM、因子模型、Alpha/Beta
第 3 步：回测框架与偏差清单
第 4 步：交易成本、滑点、冲击成本
第 5 步：因子研究：分组、IC、换手、暴露
第 6 步：组合构建与风险约束
第 7 步：订单簿、限价单、市价单、VWAP/TWAP
第 8 步：机器学习在交易中的正确使用
第 9 步：实盘前的监控、风控、kill switch
```

### 如何降低门槛

| 内容    | 降低门槛的方法                                |
| ----- | -------------------------------------- |
| 金融计量  | 先用 Python 复现收益率、回归、t 值，不急着读全书          |
| 微观结构  | 先画一个简化订单簿，模拟限价单和市价单成交                  |
| 因子模型  | 先下载 Fama-French 数据，做一次三因子回归            |
| 回测偏差  | 每写一个回测，都强制检查“是否用了未来信息”                 |
| ML 交易 | 先做 walk-forward，而不是随机 train/test split |
| 执行    | 先比较 close 成交、next open 成交、VWAP 成交结果差异  |

## 7. 正典与奇书结合：构建多视角理解

| 视角        | 代表                           | 提供什么        | 局限         |
| --------- | ---------------------------- | ----------- | ---------- |
| 主流共识视角    | CAPM、Fama-French、组合理论        | 解释收益来源和风险暴露 | 对短期交易细节不足  |
| 经典教材/论文视角 | 金融计量、市场微观结构、资产定价             | 严谨、可推导、可检验  | 实操门槛高      |
| 大师个人风格视角  | López de Prado、Chan、Ilmanen  | 很多实践坑和研究框架  | 观点有风格偏好    |
| 批判性/另类视角  | 行为金融、适应性市场、市场制度批判            | 解释市场为何非完全有效 | 不一定直接转化成策略 |
| 产业实践视角    | QuantConnect、Qlib、交易所规则、风控案例 | 告诉你系统如何落地   | 需要自己补理论    |

这几种视角互相补充：
**正典给你“什么是严谨”，奇书给你“实战中哪里会死”，产业资料给你“如何落地”。**

它们也互相冲突：
资产定价会提醒你“超额收益可能只是风险补偿”；机器学习派会说“复杂非线性关系可学习”；微观结构派会说“你以为的 alpha 可能只是无法成交的幻觉”。

## 8. Position：这个领域有哪些位置？

### 主要角色结构

| 位置              | 核心利益      | 掌握资源         | 如何影响领域       |
| --------------- | --------- | ------------ | ------------ |
| 量化研究员           | 找到可交易信号   | 数据、模型、统计能力   | 决定策略来源       |
| 量化开发            | 把研究变成系统   | 工程、架构、性能     | 决定策略能否稳定运行   |
| 交易员/执行团队        | 降低成交成本    | 订单、流动性、市场经验  | 决定纸面收益保留多少   |
| 投资组合经理          | 管理资本配置    | 风险预算、资金、组合权重 | 决定策略组合与规模    |
| 风控/合规           | 防止灾难与违规   | 权限、规则、监控     | 决定系统边界       |
| 数据供应商           | 卖数据       | 原始数据、清洗能力    | 决定研究起点       |
| 券商/Prime Broker | 提供交易通道和融资 | 交易接口、融券、杠杆   | 决定可交易工具      |
| 交易所             | 制定市场机制    | 撮合、规则、费用、数据  | 改变策略生态       |
| 监管者             | 维护市场公平稳定  | 法规、处罚、信息披露   | 约束程序化交易      |
| 普通学习者           | 建立作品与能力   | 时间、编程、研究能力   | 适合从复现和小型研究切入 |

### 普通学习者最适合的位置

在没有更多背景信息时，我建议你的切入位置是：

> **量化研究复现者 → 策略原型开发者 → 小型组合研究者**

不要一开始定位为“实盘交易员”或“高频交易者”。更好的路线是：

1. 复现经典因子
2. 写干净的回测
3. 做策略诊断
4. 建立案例库
5. 形成一份可展示的研究报告
6. 再考虑纸面交易或极小资金验证

## 9. Code：这个领域的规则和评价体系

### 这个领域真正追求的最终目标

不是“收益率最高”，而是：

> **在可解释风险、可承受回撤、可执行成本、可扩展容量和合规边界内，长期稳定地产生风险调整后收益。**

### 外部评价体系

| 指标     | 意义              |
| ------ | --------------- |
| 年化收益   | 表面结果            |
| 夏普率    | 单位波动收益          |
| 最大回撤   | 最坏痛苦程度          |
| Calmar | 收益/最大回撤         |
| 信息比率   | 相对基准的超额收益质量     |
| 换手率    | 成本压力            |
| 容量     | 能管理多少钱          |
| 稳定性    | 不同市场、年份、资产中是否有效 |
| 样本外表现  | 是否真的泛化          |
| 实盘跟踪误差 | 回测与真实交易差异       |

### 内部专业评价标准

| 专业标准  | 具体表现              |
| ----- | ----------------- |
| 假设清楚  | 为什么这个现象会存在        |
| 数据可信  | 时间戳、复权、退市、延迟都处理   |
| 回测严谨  | 无未来函数、无数据泄露       |
| 成本真实  | 手续费、滑点、冲击、融资都考虑   |
| 风险清楚  | beta、行业、风格、集中度可解释 |
| 代码可复现 | 一键运行、日志完整、参数可追踪   |
| 结果可拆解 | 知道收益来自哪里          |
| 失败可诊断 | 策略失效时能定位原因        |

### 常见伪目标

| 伪目标      | 为什么危险       |
| -------- | ----------- |
| 回测曲线漂亮   | 可能是过拟合      |
| 年化收益极高   | 可能不可成交或容量极小 |
| 参数优化后表现好 | 可能只是数据挖掘    |
| 模型很复杂    | 不等于更稳健      |
| 胜率高      | 盈亏比可能很差     |
| 论文很新     | 不等于可实盘      |
| 免费数据跑通   | 不等于专业数据可复现  |

### 看起来专业 vs 真正专业

| 看起来专业      | 真正专业               |
| ---------- | ------------------ |
| 展示收益曲线     | 展示收益分解、成本敏感性、样本外表现 |
| 说“AI 预测市场” | 说明标签、特征、验证、交易约束    |
| 堆模型        | 解释为什么这个模型适合这个市场    |
| 追求高频       | 先理解执行、滑点、撮合和监管     |
| 只讲收益       | 同时讲风险、容量和失效条件      |

一定不要把 **评价体系** 当成 **最终目标**。
夏普率、回撤、年化收益只是衡量工具，最终目标是建立一个能长期活下去的研究与交易系统。

## 13. 最终总结：你进入量化交易的最短路径

最短路径不是“学很多金融名词”，也不是“马上实盘”。

### 必须学

1. 收益率、风险、回撤、相关性
2. Alpha/Beta/因子
3. 回测偏差
4. 交易成本
5. 因子研究流程
6. 组合构建
7. 市场微观结构基础
8. 风控与监控
9. Python 数据处理
10. 一个可复现项目

### 值得学

1. 金融计量
2. 机器学习
3. 最优执行
4. 衍生品基础
5. 数据工程
6. 交易所规则
7. 机构量化流程

### 暂时不学

1. 超低延迟高频
2. FPGA
3. 强化学习实盘交易
4. 复杂期权做市系统
5. 每天追逐新策略帖子
6. 无法解释的黑箱信号

### 你的最短行动路线

```text
第 1 周：建立术语表 + 读懂量化交易流程
第 2 周：用 Python 做收益率、回撤、因子排序
第 3 周：复现一个价值/动量因子回测
第 4 周：加入交易成本、样本外验证、风险分析，写成报告
```

一句话总结：

> **先成为“能复现、能验证、能拆解风险的量化研究者”，再考虑成为“能实盘交易的人”。**
