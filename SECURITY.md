# Security Policy

## 报告范围

请报告与仓库配置、自动化流程、示例脚本、敏感信息泄露有关的问题。

不属于安全报告范围：

- 投资观点分歧。
- 策略表现争议。
- 第三方数据源或第三方平台自身的安全问题。

## 报告方式

请不要在公开 Issue 中提交 token、账号、私有数据、交易记录或漏洞细节。优先通过 GitHub 私密安全报告功能提交。

## 敏感信息规则

- 不提交 API key、交易账户、私钥、cookie、付费数据下载链接。
- 示例配置必须使用 `.env.example` 或占位符。
- 发现泄露后应立即撤销凭据，并提交清理 PR。
